 \hypertarget{md_ext_2hera_2extern_2_catch2_2docs_2deprecations_autotoc_md125}{}\doxysection{Deprecations and incoming changes}\label{md_ext_2hera_2extern_2_catch2_2docs_2deprecations_autotoc_md125}
This page documents current deprecations and upcoming planned changes inside Catch2. The difference between these is that a deprecated feature will be removed, while a planned change to a feature means that the feature will behave differently, but will still be present. Obviously, either of these is a breaking change, and thus will not happen until at least the next major release.\hypertarget{md_ext_2hera_2extern_2_catch2_2docs_2deprecations_autotoc_md126}{}\doxysubsubsection{$<$tt$>$\+Parse\+And\+Add\+Catch\+Tests.\+cmake$<$/tt$>$}\label{md_ext_2hera_2extern_2_catch2_2docs_2deprecations_autotoc_md126}
The CMake/\+CTest integration using {\ttfamily Parse\+And\+Add\+Catch\+Tests.\+cmake} is deprecated, as it can be replaced by {\ttfamily Catch.\+cmake} that provides the function {\ttfamily catch\+\_\+discover\+\_\+tests} to get tests directly from a CMake target via the command line interface instead of parsing C++ code with regular expressions.\hypertarget{md_ext_2hera_2extern_2_catch2_2docs_2deprecations_autotoc_md127}{}\doxysubsection{Planned changes}\label{md_ext_2hera_2extern_2_catch2_2docs_2deprecations_autotoc_md127}
