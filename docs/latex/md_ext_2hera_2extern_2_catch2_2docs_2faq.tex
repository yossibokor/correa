\hypertarget{md_ext_2hera_2extern_2_catch2_2docs_2faq_autotoc_md133}{}\doxysubsection{How do I run global setup/teardown only if tests will be run?}\label{md_ext_2hera_2extern_2_catch2_2docs_2faq_autotoc_md133}
Write a custom \href{event-listeners.md\#top}{\texttt{ event listener}} and place the global setup/teardown code into the {\ttfamily test\+Run$\ast$} events.\hypertarget{md_ext_2hera_2extern_2_catch2_2docs_2faq_autotoc_md134}{}\doxysubsection{How do I clean up global state between running different tests?}\label{md_ext_2hera_2extern_2_catch2_2docs_2faq_autotoc_md134}
Write a custom \href{event-listeners.md\#top}{\texttt{ event listener}} and place the cleanup code into either {\ttfamily test\+Case$\ast$} or {\ttfamily test\+Case\+Partial$\ast$} events, depending on how often the cleanup needs to happen.\hypertarget{md_ext_2hera_2extern_2_catch2_2docs_2faq_autotoc_md135}{}\doxysubsection{Why cannot I derive from the built-\/in reporters?}\label{md_ext_2hera_2extern_2_catch2_2docs_2faq_autotoc_md135}
They are not made to be overriden, in that we do not attempt to maintain a consistent internal state if a member function is overriden, and by forbidding users from using them as a base class, we can refactor them as needed later.
