\begin{DoxyItemize}
\item {\ttfamily CHECKED\+\_\+\+IF} and {\ttfamily CHECKED\+\_\+\+ELSE}
\end{DoxyItemize}

{\ttfamily CHECKED\+\_\+\+IF( expr )} is an {\ttfamily if} replacement, that also applies Catch2\textquotesingle{}s stringification machinery to the {\itshape expr} and records the result. As with {\ttfamily if}, the block after a {\ttfamily CHECKED\+\_\+\+IF} is entered only if the expression evaluates to {\ttfamily true}. {\ttfamily CHECKED\+\_\+\+ELSE( expr )} work similarly, but the block is entered only if the {\itshape expr} evaluated to {\ttfamily false}.

\begin{quote}
{\ttfamily CHECKED\+\_\+X} macros were changed to not count as failure in Catch2 X.\+Y.\+Z. \end{quote}
\doxylink{struct_example}{Example}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{int}\ a\ =\ ...;}
\DoxyCodeLine{\textcolor{keywordtype}{int}\ b\ =\ ...;}
\DoxyCodeLine{CHECKED\_IF(\ a\ ==\ b\ )\ \{}
\DoxyCodeLine{\ \ \ \ \textcolor{comment}{//\ This\ block\ is\ entered\ when\ a\ ==\ b}}
\DoxyCodeLine{\}\ CHECKED\_ELSE\ (\ a\ ==\ b\ )\ \{}
\DoxyCodeLine{\ \ \ \ \textcolor{comment}{//\ This\ block\ is\ entered\ when\ a\ !=\ b}}
\DoxyCodeLine{\}}

\end{DoxyCode}



\begin{DoxyItemize}
\item {\ttfamily CHECK\+\_\+\+NOFAIL}
\end{DoxyItemize}

{\ttfamily CHECK\+\_\+\+NOFAIL( expr )} is a variant of {\ttfamily CHECK} that does not fail the test case if {\itshape expr} evaluates to {\ttfamily false}. This can be useful for checking some assumption, that might be violated without the test necessarily failing.

\doxylink{struct_example}{Example} output\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{main.cpp:6:}
\DoxyCodeLine{FAILED\ -\/\ but\ was\ ok:}
\DoxyCodeLine{\ \ CHECK\_NOFAIL(\ 1\ ==\ 2\ )}
\DoxyCodeLine{}
\DoxyCodeLine{main.cpp:7:}
\DoxyCodeLine{PASSED:}
\DoxyCodeLine{\ \ CHECK(\ 2\ ==\ 2\ )}

\end{DoxyCode}



\begin{DoxyItemize}
\item {\ttfamily SUCCEED}
\end{DoxyItemize}

{\ttfamily SUCCEED( msg )} is mostly equivalent with {\ttfamily INFO( msg ); REQUIRE( true );}. In other words, {\ttfamily SUCCEED} is for cases where just reaching a certain line means that the test has been a success.

\begin{DoxyItemize}
\item {\ttfamily STATIC\+\_\+\+REQUIRE} and {\ttfamily STATIC\+\_\+\+CHECK}
\end{DoxyItemize}

\begin{quote}
{\ttfamily STATIC\+\_\+\+REQUIRE} was \href{https://github.com/catchorg/Catch2/issues/1362}{\texttt{ introduced}} in Catch2 2.\+4.\+2. \end{quote}
{\ttfamily STATIC\+\_\+\+REQUIRE( expr )} is a macro that can be used the same way as a {\ttfamily static\+\_\+assert}, but also registers the success with Catch2, so it is reported as a success at runtime. The whole check can also be deferred to the runtime, by defining {\ttfamily CATCH\+\_\+\+CONFIG\+\_\+\+RUNTIME\+\_\+\+STATIC\+\_\+\+REQUIRE} before including the Catch2 header.

\doxylink{struct_example}{Example}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{TEST\_CASE(\textcolor{stringliteral}{"{}STATIC\_REQUIRE\ showcase"{}},\ \textcolor{stringliteral}{"{}[traits]"{}})\ \{}
\DoxyCodeLine{\ \ \ \ STATIC\_REQUIRE(\ std::is\_void<void>::value\ );}
\DoxyCodeLine{\ \ \ \ STATIC\_REQUIRE\_FALSE(\ std::is\_void<int>::value\ );}
\DoxyCodeLine{\}}

\end{DoxyCode}


\begin{quote}
{\ttfamily STATIC\+\_\+\+CHECK} was \href{https://github.com/catchorg/Catch2/pull/2318}{\texttt{ introduced}} in Catch2 X.\+Y.\+Z. \end{quote}
{\ttfamily STATIC\+\_\+\+CHECK( expr )} is equivalent to {\ttfamily STATIC\+\_\+\+REQUIRE( expr )}, with the difference that when {\ttfamily CATCH\+\_\+\+CONFIG\+\_\+\+RUNTIME\+\_\+\+STATIC\+\_\+\+REQUIRE} is defined, it becomes equivalent to {\ttfamily CHECK} instead of {\ttfamily REQUIRE}.

\doxylink{struct_example}{Example}\+:

\DoxyCodeLine{\ \ \ \ STATIC\_CHECK(\ std::is\_void<void>::value\ );}
\DoxyCodeLine{\ \ \ \ STATIC\_CHECK\_FALSE(\ std::is\_void<int>::value\ );}
\DoxyCodeLine{\}}

\end{DoxyCode}

\begin{DoxyItemize}
\item {\ttfamily REGISTER\+\_\+\+TEST\+\_\+\+CASE}
\end{DoxyItemize}

{\ttfamily REGISTER\+\_\+\+TEST\+\_\+\+CASE( function, description )} let\textquotesingle{}s you register a {\ttfamily function} as a test case. The function has to have {\ttfamily void()} signature, the description can contain both name and tags.

\doxylink{struct_example}{Example}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{REGISTER\_TEST\_CASE(\ someFunction,\ \textcolor{stringliteral}{"{}ManuallyRegistered"{}},\ \textcolor{stringliteral}{"{}[tags]"{}}\ );}

\end{DoxyCode}


{\itshape Note that the registration still has to happen before Catch2\textquotesingle{}s session is initiated. This means that it either needs to be done in a global constructor, or before Catch2\textquotesingle{}s session is created in user\textquotesingle{}s own main.}

\item {\ttfamily DYNAMIC\+\_\+\+SECTION}
\end{DoxyItemize}

\begin{quote}
Introduced in Catch2 2.\+3.\+0. \end{quote}
{\ttfamily DYNAMIC\+\_\+\+SECTION} is a {\ttfamily SECTION} where the user can use {\ttfamily operator\texorpdfstring{$<$}{<}\texorpdfstring{$<$}{<}} to create the final name for that section. This can be useful with e.\+g. generators, or when creating a {\ttfamily SECTION} dynamically, within a loop.
